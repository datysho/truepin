# TruePin

Pinned tabs, done right: расширение Chrome, при котором закреплённую вкладку нельзя закрыть нечаянно.

Закрытие защищённой вкладки даёт нативный диалог Chrome «Leave site?». Если диалог был невозможен и вкладка закрылась тихо - расширение мгновенно возвращает её на место, закреплённой. Потерять закреплённую вкладку случайно нельзя.

Сверх защиты: закреплённые зеркалируются во все окна (копия набора в каждом окне, закрытие/закрепление/снятие синхронизируются), их наборы сохраняются и восстанавливаются из попапа, а последние 10 состояний пишутся в автосохранения. Интерфейс локализован: en, ru, uk, de, fr, es, pt, zh.

Происхождение: полная переписка с нуля идеи [chrome-lock-tab](https://github.com/gabrielmaldi/chrome-lock-tab) (Gabriel Monteagudo, лицензия WTFPL), который срабатывал через раз.

## Почему оригинал работал через раз
- Chrome показывает диалог beforeunload только после взаимодействия со страницей (sticky user activation). Вкладка после старта браузера, в которую ещё не кликали, закрывалась молча.
- Контент-скрипт не внедрялся в уже открытые вкладки после установки/обновления расширения.
- Chrome при закрытии закреплённой вкладки сначала сам снимает закрепление (`onUpdated {pinned:false}` за миллисекунды до `onRemoved`) - «автозащита закреплённых» отваливалась в самый нужный момент.
- Состояние по tab id хранилось в `storage.local` навсегда: после рестарта браузера id переиспользуются, защита липла к чужим вкладкам.
- Вкладки, выгруженные memory saver, теряли beforeunload-обработчик.

## Как работает TruePin (два слоя)
1. **Диалог подтверждения.** Контент-скрипт держит beforeunload-гард во всех вкладках (включая уже открытые - внедрение при установке). Защищённым вкладкам выключается автовыгрузка памяти (`autoDiscardable=false`), чтобы гард не умирал в фоне.
2. **Restore-net.** Расширение знает, был ли у страницы шанс показать диалог: контент-скрипт репортит user activation (включая клики в iframe). Если защищённая вкладка закрылась без этого шанса (страницу не трогали, она была выгружена, это chrome://-страница) - вкладка восстанавливается через `chrome.sessions` закреплённой на том же месте, с уведомлением.

Защита вкладки = ручной замок (клик по иконке) или «закреплена + автозащита включена». Фальшивый unpin, который Chrome делает сам в момент закрытия, отфильтрован грейс-окном 500 мс. Всё состояние - в `storage.session`: живёт при засыпании service worker, сбрасывается вместе с tab id при рестарте браузера.

## Зеркалирование: закреплённые во всех окнах
Закреплённые вкладки образуют логические группы, и каждая группа присутствует в каждом обычном окне:
- Новое окно (любое обычное: Cmd+N, «открыть в новом окне», перетаскивание) получает копию каждой закреплённой.
- Закрепил вкладку в одном окне - её копии появились во всех остальных.
- Закрыл закреплённую по-настоящему (подтвердил диалог, повторное закрытие) - её копии закрылись во всех окнах.
- Снял закрепление - вкладка осталась обычной в этом окне, копии в других закрылись.
- Копии - живые независимые экземпляры страницы: навигация внутри одной не дёргает остальные. Инкогнито не участвует. Выключается в настройках.

## Наборы закреплённых (попап по клику на иконку)
- Попап показывает текущие закреплённые (с фавиконками), переключатель замка для активной вкладки, сохранение набора по имени, список наборов и выпадающий список автосохранений.
- **Восстановление - умный дифф:** вкладки, совпавшие по адресу, остаются на месте без перезагрузки; недостающие открываются; лишние закрываются. Порядок - как в наборе. Результат зеркалируется во все окна.
- **Автосохранения - последние 10 состояний набора.** Пишутся, когда набор меняется структурно: закреплённая добавилась, удалилась или ушла на другую страницу; смена только GET-параметров/якоря записью не считается. Это и страховка (закрыл окно со всеми пинами - вернёшь), и «отменить»: перед восстановлением набора текущий состав уходит в автосохранения.
- Именованные наборы лежат в `storage.sync` - при включённой синхронизации Chrome приезжают на другие машины; автосохранения - локальные.

## Закрыть защищённую вкладку намеренно
- Подтвердить в диалоге - восстановления не будет.
- Тихо закрытая вернулась - закрыть её повторно в течение 15 с (окно настраивается).
- Снять закрепление, затем закрыть.
- Переключатель «Замок на этой вкладке» в попапе - защита выключена для этой вкладки (им же можно защитить любую незакреплённую).

## Установка
1. `chrome://extensions` - включить Developer mode.
2. Load unpacked - указать папку `extension/` этого репозитория.
3. Готово: закреплённые вкладки защищаются автоматически. Настройки - правый клик по иконке - Options.

После `git pull` с изменениями нажать Reload на карточке расширения. Если папка репозитория переехала - удалить расширение и загрузить заново с нового пути.

## Настройки
Автозащита закреплённых (вкл) - 🔒 в заголовке (вкл) - restore-net (вкл) - окно повторного закрытия (15 с) - зеркалирование во всех окнах (вкл) - автосохранения (вкл) - язык (авто по языку браузера + ручной выбор: en, ru, uk, de, fr, es, pt, zh). Ручной замок действует до конца сессии браузера.

## Честные границы
- Перезагрузка по Cmd+R/F5 проходит без вопросов (перезагрузка безопасна); кнопка Reload в тулбаре и перезагрузка из меню всё ещё спросят - браузер не даёт отличить их от закрытия заранее.
- Закрытие целого окна и Cmd+Q не перехватываются: это намеренные действия (окно возвращает Cmd+Shift+T; для выхода из Chrome включи «Warn Before Quitting»).
- На chrome://-страницах контент-скрипты запрещены - там работает только restore-net, без диалога.
- API-закрытие от другого расширения на странице со взаимодействием пройдёт без восстановления (неотличимо от подтверждённого диалога).

## Разработка
- `extension/` - MV3: `background.js` (состояние, restore-net, зеркальные группы, наборы и автосохранения), `content.js` (гард, трекинг активации, 🔒 в заголовке), `popup.*` (интерфейс), `options.*`, `i18n.js` + `_locales/` (8 языков), `icons/`.
- Тесты: `cd test && npm install && npm test` - e2e на puppeteer с реальным Chrome for Testing: 22 сценария (диалог, восстановление закреплённой, cooldown повторного закрытия, unpin, ручной замок, невмешательство в обычные вкладки, настройки, дифф-восстановление набора, триггеры автосохранений, попап, зеркалирование: новое окно/окно с URL/закрепление/закрытие/снятие, выключение зеркала, локализация, тихая пропагация закрытия без диалога, чистый лог сервис-воркера). `HEADFUL=1 npm test` - смотреть глазами.
- `test/shot.mjs <out.png>` - скриншот попапа с реальными данными (для правок UI).
- Диагностика на живом браузере: `chrome://extensions` - service worker - консоль, объект `__tpDiag` (очередь задач и трасса последних событий).
