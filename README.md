# TruePin

Pinned tabs, done right: расширение Chrome, при котором закреплённую вкладку нельзя закрыть.

Правило одно: **защищённая вкладка не закрывается**. Любое закрытие - крестиком, Cmd+W, скриптом, другим расширением - мгновенно и тихо откатывается: вкладка возвращается на место, закреплённой, с историей. Никаких диалогов и уведомлений. Хочешь закрыть - сначала сними закрепление, потом закрывай. Перезагрузка и переходы по ссылкам свободны: вкладка ведь никуда не девается.

Сверх защиты: закреплённые зеркалируются во все окна (копия набора в каждом окне), их наборы сохраняются и восстанавливаются из попапа, последние 10 состояний пишутся в автосохранения. Интерфейс локализован: en, ru, uk, de, fr, es, pt, zh.

Происхождение: идея - [chrome-lock-tab](https://github.com/gabrielmaldi/chrome-lock-tab) (Gabriel Monteagudo, WTFPL), который защищал диалогом beforeunload и срабатывал через раз: Chrome показывает такой диалог только после взаимодействия со страницей, а вкладки без взаимодействия, выгруженные из памяти и chrome:// закрывались молча. TruePin прошёл через диалоговую архитектуру (v2) и отказался от неё: восстановление закрытого надёжнее и проще предупреждений.

## Как работает
- Защита вкладки = «закреплена + автозащита включена» или ручной замок из попапа.
- Закрытие защищённой вкладки: сервис-воркер ловит `onRemoved` и возвращает вкладку через `chrome.sessions` (история, скролл и формы сохраняются); если записи в sessions нет - пересоздаёт по адресу. Ручной замок переносится на возвращённую вкладку.
- Снятие закрепления - осознанное действие: защита снимается, вкладка остаётся обычной, её копии в других окнах закрываются. Фальшивый unpin, который Chrome делает сам в момент закрытия, отфильтрован грейс-окном 500 мс.
- Всё состояние - в `storage.session`: живёт при засыпании service worker, сбрасывается вместе с tab id при рестарте браузера.

## Зеркалирование: закреплённые во всех окнах
Закреплённые вкладки образуют логические группы, и каждая группа присутствует в каждом обычном окне:
- Новое окно (любое обычное: Cmd+N, «открыть в новом окне», перетаскивание) получает копию каждой закреплённой. Заполнение ждёт, пока окно дорисует собственные вкладки (восстановление сессии), и сперва усыновляет их - дублей не бывает.
- Закрепил вкладку в одном окне - её копии появились во всех остальных.
- Закрыл копию - она вернулась на место (правило то же во всех окнах).
- Снял закрепление - вкладка осталась обычной в этом окне, копии в других закрылись.
- Копии - живые независимые экземпляры страницы: навигация внутри одной не дёргает остальные. Инкогнито не участвует. Выключается в настройках.

## Наборы закреплённых (попап по клику на иконку)
- Попап показывает текущие закреплённые (с фавиконками), переключатель замка для активной вкладки, сохранение набора по имени, список наборов и выпадающий список автосохранений.
- **Восстановление - умный дифф:** вкладки, совпавшие по адресу, остаются на месте без перезагрузки; недостающие открываются; лишние закрываются. Порядок - как в наборе. Результат зеркалируется во все окна.
- **Автосохранения - последние 10 состояний набора.** Пишутся, когда набор меняется структурно: закреплённая добавилась, удалилась или ушла на другую страницу; смена только GET-параметров/якоря записью не считается. Это и страховка (закрыл окно со всеми пинами - вернёшь), и «отменить»: перед восстановлением набора текущий состав уходит в автосохранения.
- Именованные наборы лежат в `storage.sync` - при включённой синхронизации Chrome приезжают на другие машины; автосохранения - локальные.

## Закрыть защищённую вкладку
- Закреплённая: снять закрепление, затем закрыть.
- С ручным замком: выключить «Замок на этой вкладке» в попапе, затем закрыть.
- Другого пути нет - в этом и смысл.

## Установка
1. `chrome://extensions` - включить Developer mode.
2. Load unpacked - указать папку `extension/` этого репозитория.
3. Готово: закреплённые вкладки защищаются автоматически. Настройки - правый клик по иконке - Options.

После `git pull` с изменениями нажать Reload на карточке расширения. Если папка репозитория переехала - удалить расширение и загрузить заново с нового пути.

## Настройки
Автозащита закреплённых (вкл) - 🔒 в заголовке (вкл) - зеркалирование во всех окнах (вкл) - автосохранения (вкл) - язык (авто по языку браузера + ручной выбор: en, ru, uk, de, fr, es, pt, zh). Ручной замок действует до конца сессии браузера.

## Честные границы
- Закрытие целого окна и Cmd+Q не откатываются: это действия уровня окна (окно возвращает Cmd+Shift+T, набор пинов - автосохранения и зеркала в других окнах).
- При выключенной автозащите закреплённые закрываются как обычно; закрытие копии тогда синхронизируется на все окна.
- Восстановленная вкладка получает новый tab id; для страниц без записи в `chrome.sessions` пересоздание теряет историю переходов (сама страница и адрес сохраняются).

## Разработка
- `extension/` - MV3: `background.js` (защита-реанимация, зеркальные группы, наборы и автосохранения), `content.js` (только 🔒 в заголовке), `popup.*` (интерфейс), `options.*`, `i18n.js` + `_locales/` (8 языков), `icons/`.
- Тесты: `cd test && npm install && npm test` - e2e на puppeteer с реальным Chrome for Testing: 21 сценарий (неубиваемость при всех способах закрытия, повторные закрытия, свободные перезагрузка/навигация, путь «unpin - закрыть», ручной замок с переносом на возвращённую вкладку, невмешательство в обычные вкладки, настройки, дифф-восстановление набора, триггеры автосохранений, попап и вёрстка длинных имён, зеркалирование по трём окнам, усыновление без дублей, выключение зеркала, локализация, чистый лог сервис-воркера). `HEADFUL=1 npm test` - смотреть глазами.
- `test/shot.mjs <out.png>` - скриншот попапа с реальными данными (для правок UI).
- Диагностика на живом браузере: `chrome://extensions` - service worker - консоль, объект `__tpDiag` (очередь задач и трасса последних событий).
